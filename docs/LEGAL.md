# Юридические документы на сайте

Документы для WebPay, банка и пользователей размещены в frontend и доступны без авторизации.

## Страницы

| URL | Документ |
|-----|----------|
| `/legal/terms` | Пользовательское соглашение (оферта) |
| `/legal/privacy` | Политика обработки персональных данных |
| `/legal/payment` | Условия оплаты и возврата (эскроу) |
| `/legal/requisites` | Реквизиты и контакты компании |

## Где править тексты и реквизиты

**Один файл конфигурации** (реквизиты, email, комиссия):

`frontend/src/components/Content/Legal/legalConfig.js`

Тексты разделов — в компонентах:

- `TermsOfService.jsx`
- `PrivacyPolicy.jsx`
- `PaymentRules.jsx`
- `Requisites.jsx`

Перед подключением WebPay:

1. Заполнить `legalConfig.js` (ООО/ИП, УНП, адрес, счёт, контакты).
2. Согласовать все тексты с юристом под вашу модель бизнеса.
3. Скачать чек-лист требований к сайту на [webpay.by/cards](https://webpay.by/cards/).
4. Убедиться, что ссылки на документы есть в футере и при регистрации.

## Связь с оплатой

Документ `PaymentRules` описывает:

- оплату через платёжный провайдер (карта не на нашем сайте);
- эскроу до подтверждения заказа;
- комиссию 10%;
- возврат и выплату исполнителю на IBAN.

Это соответствует логике backend (`payments_crud.py`, статусы `escrow` / `released`).
